31  /**
32   * Bootstrap class used to startup a JAX-RS application in Java SE environments.
33   * <p>
34   * The {@code SeBootstrap} class is available in a Jakarta EE container environment as well; however, support for the Java SE
35   * bootstrapping APIs is <em>not required</em> in container environments.
36   * </p>
37   * <p>
38   * In a Java SE environment an application is getting started by the following command using default configuration
39   * values (i. e. mounting application at {@code http://localhost:80/} <em>or a different port</em> (there is <em>no
40   * particular default port</em> mandated by this specification). As the JAX-RS implementation is free to choose any port
41   * by default, the caller will not know the actual port unless explicitly checking the actual configuration of the
42   * instance started:
43   * </p>
44   *
45   * <pre>
46   * Application app = new MyApplication();
47   * SeBootstrap.Configuration config = SeBootstrap.Configuration.builder().build();
48   * SeBootstrap.start(app, config).thenAccept(instance -&gt; instance.configuration().port());
49   * </pre>
50   *
51   * <p>
52   * Running instances can be instructed to stop serving the application:
53   * </p>
54   *
55   * <pre>
56   * SeBootstrap.start(app, config).thenAccept(instance -&gt; { ... instance.stop(); } );
57   * </pre>
58   *
59   * <p>
60   * A shutdown callback can be registered which will get invoked once the implementation stops serving the application:
61   * </p>
62   *
63   * <pre>
64   * instance.stop().thenAccept(stopResult -&gt; ...));
65   * </pre>
66   *
67   * {@code stopResult} is not further defined but solely acts as a wrapper around a native result provided by the
68   * particular JAX-RS implementation. Portable applications should not assume any particular data type or value.
69   *
70   * <p>
71   * Protocol, host address, port and root path can be overridden explicitly. As the JAX-RS implementation is bound to
72   * that values, no querying of the actual configuration is needed in that case:
73   * </p>
74   *
75   * <pre>
76   * SeBootstrap.Configuration.builder().protocol("HTTPS").host("0.0.0.0").port(8443).rootPath("api").build();
77   * </pre>
78   *
79   * <p>
80   * TLS can be configured by explicitly passing a customized {@link SSLContext}:
81   * </p>
82   *
83   * <pre>
84   * SSLContext tls = SSLContext.getInstance("TLSv1.2");
85   * // ...further initialize context here (see JSSE API)...
86   * SeBootstrap.Configuration.builder().protocol("HTTPS").sslContext(tls).build();
87   * </pre>
88   *
89   * <p>
90   * In case of HTTPS, client authentication can be enforced to ensure that only <em>trustworthy</em> clients can connect:
91   * </p>
92   *
93   * <pre>
94   * SeBootstrap.Configuration.builder().protocol("HTTPS").sslClientAuthentication(SSLClientAuthentication.MANDATORY).build();
95   * </pre>
96   *
97   * <p>
98   * Implementations are free to support more use cases by native properties, which effectively render the application
99   * non-portable:
100  * </p>
101  *
102  * <pre>
103  * SeBootstrap.Configuration.builder().property("productname.foo", "bar").build()
104  * </pre>
105  *
106  * <p>
107  * Bulk-loading allows to attach configuration storages easily without the need to write down all properties to be
108  * transferred. Hence, even properties unknown to the application author will get channeled into the implementation.
109  * This can be done both, explicitly (hence portable) and implicitly (hence <em>not necessarily</em> portable as no
110  * particular configuration mechanics are required to be supported by compliant implementations):
111  * </p>
112  *
113  * <pre>
114  * // Explicit use of particular configuration mechanics is portable
115  * SeBootstrap.Configuration.builder().from((name, type) -&gt; externalConfigurationSystem.getValue(name, type)).build();
116  *
117  * // Implicitly relying on the support of particular configuration mechanics by
118  * // the actual JAX-RS implementation is not necessarily portable
119  * SeBootstrap.Configuration.builder().from(externalConfigurationSystem).build();
120  * </pre>
121  *
122  * @author Markus KARG (markus@headcrashing.eu)
123  * @since 3.1
124  */

218         /**

220          * <p>
221          * A compliant implementation at least MUST accept the strings {@code "HTTP"} and {@code "HTTPS"} if these protocols are
222          * supported.

226          * </p>
227          *

232         /**
233          * Configuration key for the hostname or IP address an application is bound to.
234          * <p>
235          * A compliant implementation at least MUST accept string values bearing hostnames, IP4 address text representations,
236          * and IP6 address text representations. If a hostname string, the special IP4 address string {@code "0.0.0.0"} or

263          */

300         /**
301          * Secure socket client authentication policy
302          *
303          * <p>
304          * This policy is used in secure socket handshake to control whether the server <em>requests</em> client authentication,
305          * and whether <em>successful</em> client authentication is <em>mandatory</em> (i. e. connection attempt will fail for
306          * invalid clients).
307          * </p>
308          *
309          * @author Markus KARG (markus@headcrashing.eu)

313 
314             /**
315              * Server will <em>not request</em> client authentication.
316              *
317              * @since 3.1
318              */
319             NONE,
320 
321             /**
322              * Client authentication is performed, but invalid clients are <em>accepted</em>.
323              *
324              * @since 3.1
325              */
326             OPTIONAL,
327 
328             /**
329              * Client authentication is performed, and invalid clients are <em>rejected</em>.
330              *
331              * @since 3.1
332              */
333             MANDATORY
334         }

675          * @since 3.1

679         /**

687         /**
688          * Result of stopping the application instance.
689          *
690          * @author Markus KARG (markus@headcrashing.eu)
691          * @since 3.1
692          */
693         interface StopResult {
694 
695             /**
696              * Provides access to the wrapped native shutdown result.
697              * <p>
698              * Implementations may, or may not, have native shutdown results. Portable applications should not invoke this method,
699              * as the outcome is undefined.
700              * </p>
701              *
702              * @param <T> Requested type of the native result to return.
703              * @param nativeClass Requested type of the native result to return.
704              * @return Native result of shutting down the running application instance or {@code null} if the implementation has no
705              * native result.
706              * @throws ClassCastException if the result is not {@code null} or is not assignable to the type {@code T}.
707              * @since 3.1
708              */
709             <T> T unwrap(Class<T> nativeClass);
710         }
711 
712         /**
713          * Provides access to the wrapped native handle of the application instance.
714          * <p>
715          * Implementations may, or may not, have native handles. Portable applications should not invoke this method, as the
716          * outcome is undefined.
717          * </p>
718          *
719          * @param <T> Requested type of the native handle to return.
720          * @param nativeClass Requested type of the native handle to return.
721          * @return Native handle of the running application instance or {@code null} if the implementation has no native handle.
722          * @throws ClassCastException if the handle is not {@code null} and is not assignable to the type {@code T}.
723          * @since 3.1
724          */
725         <T> T unwrap(Class<T> nativeClass);
726 
727         /**
728          * Registers a consumer for a {@link StopResult} which will be executed in a new thread
729          * during the JVM shutdown phase.
730          *
731          * @param consumer The consumer.
732          * @since 3.1
733          */
734         default void stopOnShutdown(Consumer<StopResult> consumer) {
735             Runtime.getRuntime().addShutdownHook(new Thread(() -> stop().thenAccept(consumer)));
736         }
737     }
