17  package jakarta.ws.rs.core;
18  
19  import java.util.Map;

21  /**
22   * Represents a client or server-side configurable context in JAX-RS.
23   *
24   * A configurable context can be used to define the JAX-RS components as well as additional meta-data that should be
25   * used in the scope of the configured context. The modification of the context typically involves setting properties or
26   * registering custom JAX-RS components, such as providers and/or features. All modifications of a {@code Configurable}
27   * context are reflected in the associated {@link Configuration} state which is exposed via {@link #getConfiguration()}
28   * method.
29   * <p>
30   * A configurable context can be either indirectly associated with a particular JAX-RS component (such as application or
31   * resource method configurable context passed to a {@link Feature} or {@link jakarta.ws.rs.container.DynamicFeature}
32   * meta-providers) or can be directly represented by a concrete JAX-RS component implementing the {@code Configurable}
33   * interface (such as {@link jakarta.ws.rs.client.Client} or {@link jakarta.ws.rs.client.WebTarget}). As such, the exact
34   * scope of a configuration context is typically determined by a use case scenario in which the context is accessed.
35   * </p>
36   * <h2>Setting properties.</h2>
37   * <p>
38   * New properties can be set using the {@link #property} method. Similarly, updating a value of an existing property can
39   * be achieved using the same method. Information about the configured set of properties is available via the underlying
40   * {@code Configuration} object. An existing property can be removed by assigning a {@code null} value to the property.
41   * </p>
42   * <h2>Registering JAX-RS components.</h2>
43   * <p>
44   * Registered custom JAX-RS component classes and instances are important part of the contextual configuration
45   * information as these are the main factors that determine the capabilities of a configured runtime. Implementations
46   * SHOULD warn about and ignore registrations that do not conform to the requirements of supported JAX-RS components in
47   * a given configurable context.
48   * </p>
49   * <p>
50   * In most cases, when registering a JAX-RS component, the simplest form of methods ({@link #register(Class)} or
51   * {@link #register(Object)}) of the available registration API is sufficient.
52   * </p>
53   * <p>
54   * For example:
55   * </p>
56   *
57   * <pre>
58   * config.register(HtmlDocumentReader.class);
59   * config.register(new HtmlDocumentWriter(writerConfig));
60   * </pre>
61   * <p>
62   * In some situations a JAX-RS component class or instance may implement multiple provider contracts recognized by a
63   * JAX-RS implementation (e.g. filter, interceptor or entity provider). By default, the JAX-RS implementation MUST
64   * register the new component class or instance as a provider for all the recognized provider contracts implemented by
65   * the component class.
66   * </p>
67   * <p>
68   * For example:
69   * </p>
70   *
71   * <pre>
72   * &#64;Priority(ENTITY_CODER)
73   * public class GzipInterceptor
74   *         implements ReaderInterceptor, WriterInterceptor { ... }
75   *
76   * ...
77   *
78   * // register GzipInterceptor as a ReaderInterceptor
79   * // as well as a WriterInterceptor
80   * config.register(GzipInterceptor.class);
81   * </pre>
82   * <p>
83   * There are however situations when the default registration of a JAX-RS component to all the recognized provider
84   * contracts is not desirable. In such cases users may use other versions of the {@code register(...)} method to
85   * explicitly specify the collection of the provider contracts for which the JAX-RS component should be registered
86   * and/or the priority of each registered provider contract.
87   * </p>
88   * <p>
89   * For example:
90   * </p>
91   *
92   * <pre>
93   * &#64;Priority(USER)
94   * public class ClientLoggingFilter
95   *         implements ClientRequestFilter, ClientResponseFilter { ... }
96   *
97   * &#64;Priority(ENTITY_CODER)
98   * public class GzipInterceptor
99   *         implements ReaderInterceptor, WriterInterceptor { ... }
100  *
101  * ...
102  *
103  * // register ClientLoggingFilter as a ClientResponseFilter only
104  * config.register(ClientLoggingFilter.class, ClientResponseFilter.class);
105  *
106  * // override the priority of registered GzipInterceptor
107  * // and both of it's provider contracts
108  * config.register(GzipInterceptor.class, 6500);
109  * </pre>
110  * <p>
111  * As a general rule, for each JAX-RS component class there can be at most one registration &mdash; class-based or
112  * instance-based &mdash; configured at any given moment. Implementations MUST reject any attempts to configure a new
113  * registration for a provider class that has been already registered in the given configurable context earlier.
114  * Implementations SHOULD also raise a warning to inform the user about the rejected component registration.
115  * </p>
116  * <p>
117  * For example:
118  * </p>
119  *
120  * <pre>
121  * config.register(GzipInterceptor.class, WriterInterceptor.class);
122  * config.register(GzipInterceptor.class); // Rejected by runtime.
123  * config.register(new GzipInterceptor()); // Rejected by runtime.
124  * config.register(GzipInterceptor.class, 6500); // Rejected by runtime.
125  *
126  * config.register(new ClientLoggingFilter());
127  * config.register(ClientLoggingFilter.class); // rejected by runtime.
128  * config.register(ClientLoggingFilter.class,
129  *         ClientResponseFilter.class); // Rejected by runtime.
130  * </pre>
131  *
132  * @param <C> generic configurable Java type
133  * @author Marek Potociar
134  * @since 2.0
135  */
136 public interface Configurable<C extends Configurable> {
137 
138     /**
139      * Get a live view of an internal configuration state of this configurable instance.
140      *
141      * Any changes made using methods of this {@code Configurable} instance will be reflected in the returned
142      * {@code Configuration} instance.
143      * <p>
144      * The returned {@code Configuration} instance and the collection data it provides are not thread-safe wrt. modification
145      * made using methods on the parent configurable object.
146      * </p>
147      *
148      * @return configuration live view of the internal configuration state.
149      */
150     public Configuration getConfiguration();
151 
152     /**
153      * Set the new configuration property, if already set, the existing value of the property will be updated. Setting a
154      * {@code null} value into a property effectively removes the property from the property bag.
155      *
156      * @param name property name.
157      * @param value (new) property value. {@code null} value removes the property with the given name.
158      * @return the updated configurable instance.
159      */
160     public C property(String name, Object value);
161 
162     /**
163      * Register a class of a custom JAX-RS component (such as an extension provider or a {@link jakarta.ws.rs.core.Feature
164      * feature} meta-provider) to be instantiated and used in the scope of this configurable context.
165      *
166      * Implementations SHOULD warn about and ignore registrations that do not conform to the requirements of supported
167      * JAX-RS component types in the given configurable context. Any subsequent registration attempts for a component type,
168      * for which a class or instance-based registration already exists in the system MUST be rejected by the JAX-RS
169      * implementation and a warning SHOULD be raised to inform the user about the rejected registration.
170      *
171      * The registered JAX-RS component class is registered as a contract provider of all the recognized JAX-RS or
172      * implementation-specific extension contracts including meta-provider contracts, such as {@code Feature} or
173      * {@link jakarta.ws.rs.container.DynamicFeature}.
174      * <p>
175      * As opposed to component instances registered via {@link #register(Object)} method, the lifecycle of components
176      * registered using this class-based {@code register(...)} method is fully managed by the JAX-RS implementation or any
177      * underlying IoC container supported by the implementation.
178      * </p>
179      *
180      * @param componentClass JAX-RS component class to be configured in the scope of this configurable context.
181      * @return the updated configurable context.
182      */
183     public C register(Class<?> componentClass);
184 
185     /**
186      * Register a class of a custom JAX-RS component (such as an extension provider or a {@link jakarta.ws.rs.core.Feature
187      * feature} meta-provider) to be instantiated and used in the scope of this configurable context.
188      * <p>
189      * This registration method provides the same functionality as {@link #register(Class)} except that any priority
190      * specified on the registered JAX-RS component class via {@code jakarta.annotation.Priority} annotation is overridden
191      * with the supplied {@code priority} value.
192      * </p>
193      * <p>
194      * Note that in case the priority is not applicable to a particular provider contract implemented by the class of the
195      * registered component, the supplied {@code priority} value will be ignored for that contract.
196      * </p>
197      *
198      * @param componentClass JAX-RS component class to be configured in the scope of this configurable context.
199      * @param priority the overriding priority for the registered component and all the provider contracts the component
200      * implements.
201      * @return the updated configurable context.
202      */
203     public C register(Class<?> componentClass, int priority);
204 
205     /**
206      * Register a class of a custom JAX-RS component (such as an extension provider or a {@link jakarta.ws.rs.core.Feature
207      * feature} meta-provider) to be instantiated and used in the scope of this configurable context.
208      * <p>
209      * This registration method provides the same functionality as {@link #register(Class)} except the JAX-RS component
210      * class is only registered as a provider of the listed extension provider or meta-provider {@code contracts}. All
211      * explicitly enumerated contract types must represent a class or an interface implemented or extended by the registered
212      * component. Contracts that are not {@link Class#isAssignableFrom(Class) assignable from} the registered component
213      * class MUST be ignored and implementations SHOULD raise a warning to inform users about the ignored contract(s).
214      * </p>
215      *
216      * @param componentClass JAX-RS component class to be configured in the scope of this configurable context.
217      * @param contracts the specific extension provider or meta-provider contracts implemented by the component for which
218      * the component should be registered. Implementations MUST ignore attempts to register a component class for an empty
219      * or {@code null} collection of contracts via this method and SHOULD raise a warning about such event.
220      * @return the updated configurable context.
221      */
222     public C register(Class<?> componentClass, Class<?>... contracts);
223 
224     /**
225      * Register a class of a custom JAX-RS component (such as an extension provider or a {@link jakarta.ws.rs.core.Feature
226      * feature} meta-provider) to be instantiated and used in the scope of this configurable context.
227      * <p>
228      * This registration method provides same functionality as {@link #register(Class, Class[])} except that any priority
229      * specified on the registered JAX-RS component class via {@code jakarta.annotation.Priority} annotation is overridden for
230      * each extension provider contract type separately with an integer priority value specified as a value in the supplied
231      * map of [contract type, priority] pairs.
232      * </p>
233      * <p>
234      * Note that in case a priority is not applicable to a provider contract registered for the JAX-RS component, the
235      * supplied priority value is ignored for such contract.
236      * </p>
237      *
238      * @param componentClass JAX-RS component class to be configured in the scope of this configurable context.
239      * @param contracts map of the specific extension provider and meta-provider contracts and their associated priorities
240      * for which the JAX-RS component is registered. All contracts in the map must represent a class or an interface
241      * implemented or extended by the JAX-RS component. Contracts that are not {@link Class#isAssignableFrom(Class)
242      * assignable from} the registered component class MUST be ignored and implementations SHOULD raise a warning to inform
243      * users about the ignored contract(s).
244      * @return the updated configurable context.
245      */
246     public C register(Class<?> componentClass, Map<Class<?>, Integer> contracts);
247 
248     /**
249      * Register an instance of a custom JAX-RS component (such as an extension provider or a {@link jakarta.ws.rs.core.Feature
250      * feature} meta-provider) to be instantiated and used in the scope of this configurable context.
251      *
252      * Implementations SHOULD warn about and ignore registrations that do not conform to the requirements of supported
253      * JAX-RS component types in the given configurable context. Any subsequent registration attempts for a component type,
254      * for which a class or instance-based registration already exists in the system MUST be rejected by the JAX-RS
255      * implementation and a warning SHOULD be raised to inform the user about the rejected registration.
256      *
257      * The registered JAX-RS component is registered as a contract provider of all the recognized JAX-RS or
258      * implementation-specific extension contracts including meta-provider contracts, such as {@code Feature} or
259      * {@link jakarta.ws.rs.container.DynamicFeature}.
260      * <p>
261      * As opposed to components registered via {@link #register(Class)} method, the lifecycle of providers registered using
262      * this instance-based {@code register(...)} is not managed by JAX-RS runtime. The same registered component instance is
263      * used during the whole lifespan of the configurable context. Fields and properties of all registered JAX-RS component
264      * instances are injected with their declared dependencies (see {@link Context}) by the JAX-RS runtime prior to use.
265      * </p>
266      *
267      * @param component JAX-RS component instance to be configured in the scope of this configurable context.
268      * @return the updated configurable context.
269      */
270     public C register(Object component);
271 
272     /**
273      * Register an instance of a custom JAX-RS component (such as an extension provider or a {@link jakarta.ws.rs.core.Feature
274      * feature} meta-provider) to be instantiated and used in the scope of this configurable context.
275      * <p>
276      * This registration method provides the same functionality as {@link #register(Object)} except that any priority
277      * specified on the registered JAX-RS component class via {@code jakarta.annotation.Priority} annotation is overridden
278      * with the supplied {@code priority} value.
279      * </p>
280      * <p>
281      * Note that in case the priority is not applicable to a particular provider contract implemented by the class of the
282      * registered component, the supplied {@code priority} value will be ignored for that contract.
283      * </p>
284      *
285      * @param component JAX-RS component instance to be configured in the scope of this configurable context.
286      * @param priority the overriding priority for the registered component and all the provider contracts the component
287      * implements.
288      * @return the updated configurable context.
289      */
290     public C register(Object component, int priority);
291 
292     /**
293      * Register an instance of a custom JAX-RS component (such as an extension provider or a {@link jakarta.ws.rs.core.Feature
294      * feature} meta-provider) to be instantiated and used in the scope of this configurable context.
295      * <p>
296      * This registration method provides the same functionality as {@link #register(Object)} except the JAX-RS component

299      * component. Contracts that are not {@link Class#isAssignableFrom(Class) assignable from} the registered component

301      * </p>
302      *

304      * @param contracts the specific extension provider or meta-provider contracts implemented by the component for which
305      * the component should be registered. Implementations MUST ignore attempts to register a component class for an empty
306      * or {@code null} collection of contracts via this method and SHOULD raise a warning about such event.
307      * @return the updated configurable context.
308      */
309     public C register(Object component, Class<?>... contracts);
310 
311     /**
312      * Register an instance of a custom JAX-RS component (such as an extension provider or a {@link jakarta.ws.rs.core.Feature
313      * feature} meta-provider) to be instantiated and used in the scope of this configurable context.
314      * <p>
315      * This registration method provides same functionality as {@link #register(Object, Class[])} except that any priority
316      * specified on the registered JAX-RS component class via {@code jakarta.annotation.Priority} annotation is overridden for

333     public C register(Object component, Map<Class<?>, Integer> contracts);
